# 垃圾回收
1. 什么是垃圾回收机制
2. 垃圾是怎么产生的
3. 为什么要进行垃圾回收
4. 垃圾回收是怎样进行的
5. V8 对垃圾回收做了什么优化

程序工作过程中会产生很多 垃圾，这些垃圾是程序不用的内存或者是之前用过了，以后不会再用的内存空间，而 GC 就是负责回收垃圾的，因为他工作在引擎内部，所以对于我们前端来说，GC 过程是相对比较无感的，这一套引擎执行而对我们又相对无感的操作也就是常说的 `垃圾回收机制` 了


在实际运行过程中，比如
1. 我创建了一个对象a ，栈的内部实际就上一个引用堆内的地址，实际的对象是在堆内的，栈内只是存储了对象的引用地址。
2. 我现在把a赋值了一个新的对象，那么原有的对象就会变成垃圾，因为没有被任何变量引用了。
3. 这个时候GC就会回收这个垃圾对象的内存空间。

## 垃圾回收机制

在 V8 引擎中，有一个概念叫做可达性，就上哪些以某种方式可访问或者可用的值，它们被保证存储在内存中，反之不可达的值就会被回收。

说白了就上定期找出哪些不再用到的变量，然后回收他们的占用的内存空间。

这个过程中会涉及到一些算法策略

1. 标记清除算法
2. 引用计数算法
