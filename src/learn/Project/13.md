# 大文件上传和断点续传

## 核心思路

把一个大文件切断成多个小块（chunk），每个小块单独上传，如果上传过程中出现网络问题，只需要重新上传未完成的块，无需重新上传整个，这样可以大大提升文件上传的效率和可靠性。

## 基本实现步骤

1. 文件的上传，根据其内容生产唯一的标识（hash），用来区分不同的文件
2. 将文件切成多个小块（chunk）带上唯一标识（MD5等），每个块单独上传，服务器端根据唯一标识和块的索引来存储每个块
3. 逐块上传，每个分片单独上传，支持并发上传，提升效率
4. 记录上传进度，前端要记录号已上传完成的分片信息，断网后可恢复上传
5. 文件合并，所有分片上传完成后，服务器端根据唯一标识和块的索引来合并成完整文件
