

# 网络安全

## xss 跨站脚本攻击

指的是攻击者通过一切办法将可执行的代码注入到网页中

例如： 我们在登录一个网站之后，一般会把登录状态保存在cookie中，当我们去访问另一个网站的时候，就会读取到cookie

xss 危害 
1. 利用虚假输入表单骗取用户个人信息。
2. 利用脚本窃取用户cookie，在被害者不知情的情况下，帮助攻击者发送恶意请求。
3. 显示伪造内容诱导用户

### 防御措施

1. 输入验证
2. 输出编码 防止参入脚本标签
3. 给cookie 设置 httpOnly
4. CSP Content security policy  建立白名单，告诉浏览器哪些外部资源可以加载和执行 `Content-Security-Policy: default-src 'self'`
5. 避免使用 v-html 
6. 避免拼接 html
7. 输入验证  对用户输入进行验证，防止注入恶意代码
8. 避免使用 eval 函数


## CSRF 跨站请求伪造

攻击流程
1. 受害者登录 a.com，并保留了登录凭证（Cookie）
2. 攻击者引诱受害者访问了b.com
3. b.com 向 a.com 发送了一个请求：a.com/act=xx浏览器会默认携带a.com的Cookie
4. a.com接收到请求后，对请求进行验证，并确认是受害者的凭证，误以为是受害者自己发送的请求
5. a.com以受害者的名义执行了act=xx
6. 攻击完成，攻击者在受害者不知情的情况下，冒充受害者，让a.com执行了自己定义的操作

### 防御措施

1. 禁止第三方网站携带Cookie  Cookie中设置 `sameSite`

2. 验证码

3. 同源检测
